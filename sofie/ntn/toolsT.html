Nordic Treebank Network:
Treebank tools

Purpose

The purpose of this page is to document the tools used in the treebank projects of the Nordic Treebank Network for creating, using and evaluating treebanks.

Tools

The table below lists the tools submitted by members of the Nordic Trebank Network. Each tool is classified with one or more letters, depending on the supported uses of the program:

  • Aa = alignment: The program can be used for aligning parallel treebanks automatically.
    • A = word alignment
    • a = sentence alignment only
  • C = corpora: The program can be used for collecting corpora in the form of text, speech, or video.
  • Ee = editing: The program can be used for editing treebanks manually.
    • E = editing of syntax graphs and semantics
    • e = editing of word annotations only
  • Ll = learning: The program can learn grammars automatically from a treebank or a corpus.
    • L = learning grammars used for syntactic and/or semantic analysis
    • l = learning grammars used for tagging only
  • Mmμ = multi-language parallel treebanks: The program can be used to visualize and manually edit aligned parallel treebanks.
    • M = word-aligned parallel treebanks
    • m = sentence-aligned parallel treebanks
    • μ = aligned parallel corpora only
  • Pp: The program can be used to parse or tag corpora.
    • P = syntactic and/or semantic parsing
    • p = morphological tagging only
  • Q: The program can be used for quantitative analysis of corpora.
  • Ss: The program can be used to search a treebank.
    • S = the query language can deal with syntax graphs
    • s = the query language deals with word sequences only
  • T: The program can be used to manipulate sound files and transscriptions.
  • V: The program can be used to visualize treebanks.
  • X: The program can be used to convert between different treebank formats.

Dangram P CG/PSG parser for Danish text. Unrestricted input. Produces syntax trees from running text. Form & function annotation. Slanttree treebank-viewer V Java programme that allows to graphically inspect, fold/unfold, and otherwise manipulate syntactic trees. Interactive tree-building and -labelling. For live text: en, da, po, fr, es, de. For teaching: 22 languages. Phrase Structure Grammar, enriched with function and form labels, as well as head/dependent marking. vislcg rule compiler P Compiles Constraint Grammer rules, that parse running text into a treebank-precursor format. Typically used to produce word based PoS and syntactic annotation, flat dependency, case roles, named entity recognition etc. Allows sets as targets, as well as correction rules. Constraint Grammar. No restrictions on linguistic representation other than it has to be expressed in a word/token based fashion. Import: (a) cg2 type rule file, (b) Running text, xml-lines allowed. Linux or Windows Open Source All Methodologically speaking, Constraint Grammar, with add-ons Constituent Grammar and Dependency Grammar. Arboretum (Danish constituent and dependency treebank), Arboratoire (French treebank), VISL teaching treebanks (cross language), Floresta Sintá(c)tica (Portuguese Treebank)
vislpsg rule compiler P Compiles phrase structure rules into grammars that parse Constraint Grammar output into treebanks. Allows syntactic functions (i.e. subject, free adverbial, ...) as terminals, rather than words. Constraint Grammar or other word based annotation as input, Constituent Grammar as output, but has also been used, e.g. for Diederichsen Field analysis. Handles discontinuity. Usually used for sentences rather than discourse. Input format: Constraint Grammar or other word based annotation. Output format: VISL treebank format, TIGER-xml (with filter) Perl (Linux or Windows) Open Source All Constraint Grammar input, Constituent Grammar output Arboretum (Danish constituent and dependency treebank), Arboratoire (French treebank), Arborest (Estonian treebank, pilot study only)
VISL Dependency Parser P (V) Constructs dependency-like representations from VISL CG analyses by specifying the Constraint Grammar left-or-right dependencies to pointers to the exact head. Simple dependency relations Input format: Ordinary text sent through the VISL CG. Output format: a) VISL treebank format, b) CG-style word-by-word tagging with word-index and relative and absolute dependencies c) Matthias Kromann's DTAG graphs, d) ASCII dependency trees Linux or Windows, runs over internet or locally Open Source Danish (may be extendible) Constraint Grammar, dependency grammar Arboretum (Danish constituent and dependency treebank), search for indentifiers starting with 'dep'

Submission guidelines

To have a new tool added to the list, please submit the following information to Matthias T. Kromann (mtk@id.cbs.dk). The deadline is November 30, 2003.

  • Tool: The name of the tool and a link to a web page describing the tool.
  • Category: A list of all the categories that the tool belongs to (see list above; if you feel an important category is missing, please send me your proposal for a new category).
  • Features (at most 500 characters): A short description of the tasks that can be performed by the tool.
  • Linguistic representation (at most 500 characters): A description of the linguistic representations that are supported by the tool. If relevant, please indicate whether the representation supports:
    • phrase-structure/dependency-structure
    • node attributes/feature structures/edge labels
    • text/sound/transcriptions
    • single speaker/multiple speakers
    • trees/general graphs
    • cyclicity/acyclicity
    • continuity/discontinuity
    • sentence-based/entire discourse
  • Import/export formats: A list of all the file formats (with links to detailed descriptions) which the tool can use for import/export.
  • Platform: Special requirements with respect to operating system and installed software.
  • License: How the program is licensed (eg, "free", "commercial", "open source", "web application")
  • Supported languages: A list of all supported languages ("All" if the tool is language-independent).
  • Supported linguistic theories: A list with names of the most important linguistic theories (with links to more documentation) that are supported by the tool.
  • Treebank projects using tool: A list with names of treebank projects using the tool (with links to each project).
The information should be submitted as text, optionally including hand-written HTML markup, as below:
  • bold: <b>...text...</b>
  • italic: <b>...text...</i>
  • link: <a href="...url...">...text...</a>
