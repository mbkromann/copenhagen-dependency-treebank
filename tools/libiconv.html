Introduction to libiconv

For historical = reasons,=20 international text is often encoded using a language or country = dependent=20 character encoding. With the advent of the internet and the frequent = exchange of=20 text across countries - even the viewing of a web page from a foreign = country is=20 a "text exchange" in this context -, conversions between these encodings = have=20 become important. They have also become a problem, because many = characters which=20 are present in one encoding are absent in many other encodings. To solve = this=20 mess, the Unicode encoding has been created. It is a super-encoding of = all=20 others and is therefore the default encoding for new text formats like = XML.=20

Still, many computers still operate in locale with a traditional = (limited)=20 character encoding. Some programs, like mailers and web browsers, must = be able=20 to convert between a given text encoding and the user's encoding. Other = programs=20 internally store strings in Unicode, to facilitate internal processing, = and need=20 to convert between internal string representation (Unicode) and external = string=20 representation (a traditional encoding) when they are doing I/O. GNU = libiconv is=20 a conversion library for both kinds of applications.=20

Details

This library provides an iconv() = implementation,=20 for use on systems which don't have one, or whose implementation cannot = convert=20 from/to Unicode.=20

It can convert from any of these = encodings=20 to any other, through Unicode conversion.=20

It has also some limited support for transliteration, i.e. when a = character=20 cannot be represented in the target character set, it can be = approximated=20 through one or several similarly looking characters. Transliteration is=20 activated when "//TRANSLIT" is appended to the target encoding name.=20

libiconv is for you if your application needs to support multiple = character=20 encodings, but that support lacks from your system.=20

Installation

As usual for GNU packages: 
$ ./configure =
--prefix=3D/usr/local
$ make
$ make install

After installing GNU libiconv for the first time, it is recommended = to=20 recompile and reinstall GNU gettext, so that it can take advantage of = libiconv.=20

On systems other than GNU/Linux, the iconv program will be = internationalized=20 only if GNU gettext has been built and installed before GNU libiconv. = This means=20 that the first time GNU libiconv is installed, we have a circular = dependency=20 between the GNU libiconv and GNU gettext packages, which can be resolved = by=20 building and installing either=20

  • first libiconv, then gettext, then libiconv again,
or = (on systems=20 supporting shared libraries, excluding AIX)=20
  • first gettext, then libiconv, then gettext again.
Recall = that=20 before building a package for the second time, you need to erase the = traces of=20 the first build by running "make distclean".=20

This library can be built and installed in two variants:=20

  • The library mode. This works on all systems, and uses a library=20 libiconv.so and a header file = <iconv.h>. (Both=20 are installed through "make install".)=20

    To use it, simply #include <iconv.h> and use the = functions.=20

    To use it in an autoconfiguring package:=20

    • If you don't use automake, append m4/iconv.m4 to = your=20 aclocal.m4 file.=20
    • If you do use automake, add m4/iconv.m4 to your m4 = macro=20 repository.=20
    • Add to the link command line of libraries and executables that = use the=20 functions the placeholder @LIBICONV@ (or, if using = libtool for=20 the link, @LTLIBICONV@). If you use automake, the right = place=20 for these additions are the *_LDADD variables.
    Note that=20 iconv.m4 is also part of the GNU gettext package, which = installs=20 it in /usr/local/share/aclocal/iconv.m4.=20

  • The libc plug/override mode. This works on GNU/Linux, Solaris and = OSF/1=20 systems only. It is a way to get good iconv support without having = glibc-2.1.=20 It installs a library preloadable_libiconv.so. This = library can=20 be used with LD_PRELOAD, to override the iconv* functions present in = the C=20 library.=20
    • On GNU/Linux and Solaris: 
      $ export =
LD_PRELOAD=3D/usr/local/lib/preloadable_libiconv.so
    • On OSF/1: 
      $ export =
_RLD_LIST=3D/usr/local/lib/preloadable_libiconv.so:DEFAULT
      • A=20 program's source need not be modified, the program need not even be=20 recompiled. Just set the LD_PRELOAD environment variable, that's it! =
