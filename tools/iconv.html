NAME

iconv - character set = conversion

SYNOPSIS

iconv [OPTION...] [-f =
encoding] [-t encoding] [inputfile ...]
iconv -l

DESCRIPTION

The iconv program converts text from one = encoding=20 to another encoding. More precisely, it converts from the = encoding=20 given for the -f option to the encoding given for = the=20 -t option. Either of these encodings defaults to the = encoding of=20 the current locale. All the inputfiles are read and = converted in=20 turn; if no inputfile is given, the standard input is used. = The=20 converted text is printed to standard = output.
The encodings permitted are system dependent. For = the=20 libiconv implementation, they are listed in the iconv_open(3) = manual=20 page.
Options controlling the input and output=20 format:
-f encoding,=20 --from-code=3Dencoding
Specifies the encoding of the = input.
-t encoding,=20 --to-code=3Dencoding
Specifies the encoding of the = output.
Options controlling conversion=20 problems:
-c
When this option is given, characters that cannot = be=20 converted are silently discarded, instead of leading to a = conversion=20 error.
--unicode-subst=3Dformatstring
When this option is given, Unicode characters that = cannot=20 be represented in the target encoding are replaced with a = placeholder=20 string that is constructed from the given formatstring, = applied to=20 the Unicode code point. The formatstring must be a format = string in=20 the same format as for the printf command or the = printf()=20 function, taking either no argument or exactly one unsigned = integer=20 argument.
--byte-subst=3Dformatstring=
When this option is given, bytes in the input that = are not=20 valid in the source encoding are replaced with a placeholder = string that=20 is constructed from the given formatstring, applied to the = byte's=20 value. The formatstring must be a format string in the same = format=20 as for the printf command or the printf() function, = taking=20 either no argument or exactly one unsigned integer=20 argument.
--widechar-subst=3Dformatstring
When this option is given, wide characters in the = input=20 that are not valid in the source encoding are replaced with a = placeholder=20 string that is constructed from the given formatstring, = applied to=20 the byte's value. The formatstring must be a format string = in the=20 same format as for the printf command or the = printf()=20 function, taking either no argument or exactly one unsigned = integer=20 argument.
Options controlling error = output:
-s, = --silent
When this option is given, error messages about = invalid or=20 unconvertible characters are omitted, but the actual converted = text is=20 unaffected.
The iconv -l or iconv --list command = lists=20 the names of the supported encodings, in a system dependent = format. For=20 the libiconv implementation, the names are printed in upper case,=20 separated by whitespace, and alias names of an encoding are listed = on the=20 same line as the encoding itself.

EXAMPLES

iconv -f ISO-8859-1 -t = UTF-8
converts input from the old West-European encoding = ISO-8859-1 to Unicode.
iconv -f KOI8-R =
--byte-subst=3D"<0x%x>"
                --unicode-subst=3D"<U+%04X>"
converts input from the old Russian encoding = KOI8-R to the=20 locale encoding, substituting an angle bracket notation with = hexadecimal=20 numbers for invalid bytes and for valid but unconvertible=20 characters.
iconv --list
lists the supported = encodings.
